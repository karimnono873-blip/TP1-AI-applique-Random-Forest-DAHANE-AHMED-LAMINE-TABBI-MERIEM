1. Problématique et Objectifs

Une entreprise souhaite classer automatiquement ses clients pour adapter sa stratégie marketing[cite: 208]. La problématique s'inscrit dans le cadre de la classification supervisée en utilisant l'algorithme des forêts aléatoires (Random Forest)[cite: 209].

Le jeu de données utilisé est le Mall Customers Dataset[cite: 210]. Ce dataset contient des informations démographiques et comportementales sur des visiteurs de centre commercial[cite: 4]. Pour appliquer un algorithme supervisé, il est impératif de labelliser les données[cite: 213]. L'objectif principal de ce TP est de créer trois classes distinctes de consommateurs basées sur leurs revenus annuels[cite: 214], puis d'entraîner un modèle capable de prédire cette classe en fonction de l'âge et du score de dépense.

2. Notions Fondamentales

L'Algorithme Random Forest

Le Random Forest est un modèle d'Ensemble Learning basé sur le Bagging[cite: 24, 39]. L'idée générale est de combiner plusieurs modèles (ici, des arbres de décision) pour obtenir un modèle plus robuste et plus précis[cite: 21]. Chaque arbre est construit sur un échantillon aléatoire des données (avec remise, appelé Bootstrap) et un sous-ensemble aléatoire de variables[cite: 24, 115, 116, 117, 118]. L'agrégation des prédictions se fait par vote majoritaire pour la classification[cite: 119, 120]. Cette approche permet de réduire la variance [cite: 46] et de limiter les risques de surapprentissage (overfitting)[cite: 166].

Déséquilibre des Classes (Imbalanced Data)

En créant des catégories de revenus, il est fréquent qu'une classe contienne beaucoup plus d'individus qu'une autre. Un modèle entraîné sur des données déséquilibrées aura tendance à favoriser la classe majoritaire. Pour y remédier, nous utilisons des techniques d'équilibrage comme SMOTE (Synthetic Minority Over-sampling Technique) qui génère des données synthétiques pour la classe minoritaire, ou l'utilisation de poids (Class Weights) dans l'algorithme.

Matrice de Confusion & Rapport de Classification

La Matrice de Confusion permet de visualiser les performances du modèle en croisant les valeurs réelles et les prédictions (Vrais Positifs, Faux Positifs, etc.). Le Rapport de Classification regroupe des métriques clés dérivées de cette matrice :

  • Précision (Precision) : Sur toutes les prédictions positives faites par le modèle, combien étaient réellement correctes ?
  • Rappel (Recall) : Sur tous les cas réels positifs, combien le modèle a-t-il réussi à identifier ?
  • F1-Score : La moyenne harmonique de la précision et du rappel, idéale pour les bases de données déséquilibrées.

3. Résolution et Code Python

Voici l'implémentation complète pour traiter les données du dataset Mall_Customers.csv, appliquer la labellisation, équilibrer les données et évaluer le modèle Random Forest.

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report, confusion_matrix
from imblearn.over_sampling import SMOTE

# 1. Chargement des données
df = pd.read_csv('Mall_Customers.csv')

# 2. Labellisation : Création de 3 classes selon le revenu (Annual Income)
# On utilise les quantiles pour diviser les clients en 3 groupes (Faible, Moyen, Élevé)
def categoriser_revenu(revenu):
    if revenu < 40:
        return 'Faible'
    elif revenu <= 70:
        return 'Moyen'
    else:
        return 'Élevé'

df['Income_Class'] = df['Annual Income (k$)'].apply(categoriser_revenu)

# 3. Sélection des caractéristiques (Features) et de la cible (Target)
# Nous prédisons la classe de revenu en fonction de l'Âge et du Spending Score
X = df[['Age', 'Spending Score (1-100)']]
y = df['Income_Class']

# 4. Traitement du déséquilibre des données avec SMOTE
smote = SMOTE(random_state=42)
X_resampled, y_resampled = smote.fit_resample(X, y)

# 5. Séparation en données d'entraînement et de test (70% Train, 30% Test)
X_train, X_test, y_train, y_test = train_test_split(
    X_resampled, y_resampled, test_size=0.3, random_state=42
)

# 6. Initialisation et entraînement du modèle Random Forest
rf_model = RandomForestClassifier(
    n_estimators=100, 
    max_depth=10, 
    random_state=42
)
rf_model.fit(X_train, y_train)

# 7. Prédictions sur l'ensemble de test
y_pred = rf_model.predict(X_test)

# 8. Évaluation : Matrice de confusion et Rapport de classification
print("Rapport de Classification :\n", classification_report(y_test, y_pred))

# Visualisation de la matrice de confusion
conf_matrix = confusion_matrix(y_test, y_pred)
plt.figure(figsize=(8, 6))
sns.heatmap(conf_matrix, annot=True, fmt="d", cmap="Purples", 
            xticklabels=rf_model.classes_, yticklabels=rf_model.classes_)
plt.title("Matrice de Confusion - Random Forest")
plt.xlabel("Classe Prédite")
plt.ylabel("Classe Réelle")
plt.show()

A. Explication du fonctionnement global

Le code suit le cycle classique du Machine Learning. Nous commençons par charger le fichier contenant les 200 enregistrements[cite: 6]. L'étape cruciale est la création de la fonction categoriser_revenu() qui agit comme notre mécanisme de labellisation.
